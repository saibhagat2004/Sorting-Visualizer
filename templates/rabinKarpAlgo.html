Rabin-Karp algorithm

The Rabin-Karp algorithm is a string-searching algorithm that efficiently finds occurrences of a pattern within a given text. Unlike the naive string matching algorithm, which checks every character one by one, Rabin-Karp uses hashing to filter out substrings that do not match the pattern.

In Rabin-Karp algorithm,

  • Choose a suitable prime number (p) as the modulus to avoid overflow issues.
  • Select a base b (often the size of the character set, e.g., 256 for ASCII characters).
  • Initialize the hash value to 0.
  • Calculate the initial hash value for the pattern by iterating over each character in the pattern.
  • Slide the pattern over the text and update the hash value for each subsequent substring.
  • The hash value is calculated using a rolling hash function.
  • This function allows efficient updates to the hash value for a new substring by removing the contribution of the old character and adding the contribution of the new character.

  • Its average-case and best-case time complexity is O(n + m)

  • Its worst-case time complexity is O(nm)
